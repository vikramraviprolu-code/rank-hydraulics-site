# SEO Without Project Case Studies Design

## Goal

Improve Rank Hydraulics' search visibility while the client cannot provide detailed project case studies for roughly three months.

## Design

Shift the SEO emphasis from unverified project narratives to buyer-intent utility:

- Strengthen each product page with a clear "when to use this page" guide.
- Add industry/sector relevance that uses approved customer-logo context without claiming project outcomes.
- Keep WhatsApp and quote CTAs close to the new buyer guidance.
- Preserve careful supplier wording; logos identify referenced supplier lines, not authorization proof.
- Keep the project references public but avoid adding unsupported detail.

## Pages In Scope

- `products.html`
- `hydraulic-hose-assemblies.html`
- `stauff-hydraulic-components.html`
- `power-transmission-belts.html`
- `groz-tools.html`
- `veedol-lubricants.html`
- `industrial-coatings-welding.html`
- `project-vmi-supply.html`
- `index.html`

## Success Criteria

- Visitors can understand which page fits their requirement without needing a case study.
- Product pages include useful sector language for paper, steel, energy, ports, construction and process plants.
- Public claims stay source-backed and do not invent project descriptions.
- Static QA passes, and the changed CSS is cache-busted for GitHub Pages.
